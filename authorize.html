Authorize Endpoint

Introduction

The authorize endpoint is used to authenticate an enduser. It only supports the GET method.

Specifications

  • OIDC 1.0
  • OAuth 2.1

Request

Parameter Data type Description Example
scope space delimited string Used to assess the usage of the access token. Must include openid. openid identityprovider:userinfo weather:write
id_token_hint string Used to identity a session and an end user. ey
prompt space delimited string Used to assess the intention of the request, and can contain multiple values. login consent
client_id string Used to identify a client. 8BDB0EC9-32C0-4324-B836-B41538471A8E
redirect_uri string Used to redirect the response back to the client. https://localhost:5001/callback
code_challenge string Used when performing a request with PKCE. bairbglabvirevbir
code_challenge_method string Used to assess the method when creating the code_challenge. S256
response_type string Used to assess how to redirect data back to the clint. form_post
nonce string Used by the client and identity provider to deny reprocessing attacks. niasnbviubvisudvb
max_age integer with a minimum value of 0 Used to assess the maximum age of an authentication in seconds. 3600
state string Used to maintain state between the authorize request and the callback request received from the identity provider. nisdbvisdubvisdbvsiuv
