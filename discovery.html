Discovery Endpoint

Introduction

Discovery is used for relying parties to fetch information about the authorization server from a well known uri. Information includes endpoints and supported features.

Specifications

  • JWA
  • OAuth Discovery
  • OIDC Discovery

Discovery

An endpoint used to provide metadata about the authorization server. It exposes data such as endpoints, supported grants and algorithms used for signing tokens.

JWK

The array always returns three keys. The first is the newest key which has expired. This is important for resources which have received a JWT which is signed by an expired key. The second JWK is the currently used for signing JWT. The third JWK is the next in line, when the current JWK expires.

The format of a JWK is as follows:


{
    "kty": "RSA",
    "use": "sig",
    "kid": 1,
    "alg": "RS256",
    "n": "MODULUS VALUE",
    "e": "AQAB"
}
Property Description
kty Used to identify the type of the JWK
use Used to identity the usage of the JWK
kid Used as an identifier for the JWK
alg Used to identify the used algorithm for signing JWT
n Modulus of the RSA public key. It is Base64 Url encoded
e Exponent of the RSA public key. it is Base64 Url encoded
