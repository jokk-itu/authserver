Discovery Endpoint

Introduction

Discovery is used for relying parties to fetch information about the authorization server from a well known uri. Information includes endpoints and supported features.

Specifications

  • JWA
  • OAuth Discovery
  • OIDC Discovery

Discovery

An endpoint used to provide metadata about the authorization server. It exposes data such as endpoints, supported grants and algorithms used for signing tokens. The following is available in the metadata document:
issuer
The baseurl to the identity provider e.g: https://idp.authserver.dk

authorization_endpoint
Absolute url to the authorize endpoint e.g: https://idp.authserver.dk/connect/authorize

token_endpoint
Absolute url to the token endpoint e.g: https://idp.authserver.dk/connect/token

userinfo_endpoint
Absolute url to the userinfo endpoint e.g: https://idp.authserver.dk/connect/userinfo

jwks_uri
Absolute url to the jwks document e.g: https://idp.authserver.dk/.well-known/jwks

registration_endpoint
Absolute url to the registration endpoint e.g: https://idp.authserver.dk/connect/register

end_session_endpoint
Absolute url of the endsession endpoint e.g: https://idp.authserver.dk/connect/end-session

scopes_supported
JSON array of supported scopes defined by the IdP. For example openid and profile. Custom scopes used by protected resources are not shown.

response_types_supported
JSON array of supported response_type values, used in the authorize endpoint. "code" is the only supported value.

response_modes_supported
JSON array of supported response_mode values, used in the authorize endpoint. "form_post" is the only supported value.

grant_types_supported
JSON array of supported grant_type values, used in the token endpoint. "authorization_code", "client_credentials" and "refresh_token" are supported.

subject_types
JSON array of supported subject identifier types. "public" is supported.

id_token_signing_alg_values_supported
JSON array of supported alg values used for signing id tokens. "RS256" is supported.

token_endpoint_auth_methods_supported
JSON array of supported client authentication methods at the token endpoint. "client_secret_basic" and "client_secret_post" are supported.

token_endpoint_auth_signing_alg_values_supported
JSON array of supported alg methods for signing tokens issued at the token endpoint. "RS256" is supported.

revocation_endpoint
Absolute url to the revocation endpoint e.g: https://idp.authserver.dk/connect/revoke

revocation_endpoint_auth_methods_supported
JSON array of supported client authentication methods at the revocation endpoint. "client_secret_basic" and "client_secret_post" are supported. introspection_endpoint
Absolute url to the introspection endpoint e.g: https://idp.authserver.dk/connect/introspect

introspection_endpoint_auth_methods_supported
JSON array of supported client authentication methods at the introspection endpoint. "client_secret_basic" and "client_secret_post" are supported.

code_challenge_methods_supported
JSON array of supported alg values used for PKCE. "R256" is supported.

claims_supported
JSON array of supported claims, which are found in id tokens and userinfo endpoint. Claims include name, given_name, family_name, phone, email, address etc.

JWK

The array always returns three keys. The first is the newest key which has expired. This is important for resources which have received a JWT which is signed by an expired key. The second JWK is the currently used for signing JWT. The third JWK is the next in line, when the current JWK expires.
The format of a JWK is as follows: 
{
    "kty": "RSA",
    "use": "sig",
    "kid": 1,
    "alg": "RS256",
    "n": "MODULUS VALUE",
    "e": "AQAB"
}
