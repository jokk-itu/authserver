UserInfo Endpoint

Introduction

An endpoint to get claims about a user.

Specifications

  • OpenId Connect 1.0

UserInfo

The endpoint accepts GET and POST methods. It requires an Authorization header with an issued access token. The access token must contain the identityprovider:userinfo scope. The response is a json body of claims, which are identified from the sub claim in the access token. 
    
sequenceDiagram
participant OpenIDProvider as OP
participant RelyingParty as RP
RelyingParty->>OpenIDProvider: Get request userinfo
OpenIDProvider->>RelyingParty: 200 response userinfo

Example of a GET request:
    GET /connect/userinfo HTTP/1.1
    Host: idp.authserver.dk
    Authorization: Bearer SlAV32hkKG

Example of a POST request:
    POST /connect/userinfo HTTP/1.1
    Host: idp.authserver.dk
    Authorization: Bearer SlAV32hkKG

Example of a successful response:
    HTTP/1.1 200 OK
    Content-Type: application/json
  
    {
     "sub": "248289761001",
     "name": "Jane Doe",
     "given_name": "Jane",
     "family_name": "Doe",
     "email": "janedoe@example.com"
    }

Example of an error response:
    HTTP/1.1 401 Unauthorized
    WWW-Authenticate: error="invalid_token",
      error_description="The Access Token expired"
